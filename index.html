什么是柯里化与反柯里化

发表于 2023-05-05

柯里化
柯里化（Currying）是一种处理多元函数的方法。它产生一系列连锁函数，其中每个函数固定部分参数，并返回一个新函数，用于传回其它剩余参数的功能。柯里化的概念在函数式编程(Functional Programming)中经常能看到，并且是很多高级编程语言拥有的特性，比如JavaScript和Scala
柯里化的应用有以下好处：
参数复用
提前返回
延迟计算/运行
柯里化可以简化代码实现
在JavaScript中，可以通过函数的call和apply方法来实现柯里化。下面是一个简单的柯里化函数，用于将两个参数相加：
function curriedAdd(x) {
return function(y) {
return x + y;
};
}

const addFive = curriedAdd(5);
console.log(addFive(3)); // Output: 8
这个函数将两个参数相加，但是通过柯里化将第一个参数“5”作为一个固定参数，返回一个新函数，新函数会将第二个参数与“5”相加。这样可以在之后的代码中多次使用“5”这个参数，而不必每次都写入
柯里化的实现原理可以分为以下两个步骤：

创建一个闭包，用于存储已经传入的参数。
返回一个新函数，用于传入剩余的参数，并执行原函数。
下面是一个通用的柯里化函数实现，可以用于任意元函数的柯里化：
function currying(fn) {
return function curried() {
var args = [].slice.call(arguments),
context = this
return args.length >= fn.length ?
fn.apply(context, args) :
function () {
var rest = [].slice.call(arguments)
return curried.apply(context, args.concat(rest))
}
}
}
这个函数接收一个函数作为参数，返回一个新函数。新函数会接收任意数量的参数，并将这些参数保存到一个数组中。如果参数的数量大于或等于原函数的参数数量，那么就执行原函数并返回结果。否则，返回一个新函数，用于传入剩余的参数并且递归调用curried函数。

在柯里化函数的实现中，可以通过fn.length属性获取到原函数的参数数量，从而实现一个通用的柯里化函数。但是对于参数数量不确定的函数，可以在柯里化转换的同时，指定目标参数长度，用于替代length的作用。下面是一个适配参数不固定的函数的currying函数实现：function currying(fn, len) {
return function curried() {
var args = [].slice.call(arguments),
context = this
var _len = fn.length || len || 0
return args.length >= _len ?
fn.apply(context, args) :
function () {
var rest = [].slice.call(arguments)
return curried.apply(context, args.concat(rest))
}
}
}
这个函数接收两个参数，一个是需要柯里化的函数，另一个是指定的目标参数长度。在返回的新函数中，如果原函数的参数长度大于0，则使用原函数的参数长度；否则，使用指定的目标参数长度。这样就可以适配参数不固定的函数

1.什么是重绘和重排
重绘：当页面元素的样式改变不影响布局时，浏览器重新对元素
进行更新的过程叫做重绘。
重排：当页面元素的尺寸、结构、或某些属性发生改变时，浏览
器重新渲染部分或全部文档的过程叫做重排也叫做回流。
2.this理解
箭头函数内不存在this，沿用上一级的，上一级没有继续往上
找，直到找到有this的地方
this是函数调用的时候确定的。不是定义时候确定的
对象的方法内部, 看方法被谁调用
3.改变this有哪些方法-箭头函数特点
箭头函数
call（） apply（） bind（）
4.原型的理解
每一个构造函数都有一个 prototype 属性，指向另一个对象，所
以我们也称为原型对象
5.原型链的理解
_proto_对象原型的意义就在于为对象成员查找机制提供一个方
向，或者说一条路线

8.cookie 、本地存储、session的区别
1 存储大小不同
cookie： 4K
session : 5M
local：5M
2 数据有效期不同，
cookie可以设置， 设置过期时间， 也可以手动清空
session：仅在浏览器关闭前有效
local：永久有效，除非手动清除
3 作用域，
cookie在所有同源窗口共享
session仅在一个浏览器共享，不同浏览器不共享
local在所有同源窗口共享
4 通信，
cookie: 通过http,在浏览器和服务器传递
session仅在客户端
local仅在客户端
5 应用，
cookie判断用户是否登陆过，记忆密码自动登录
session敏感账号一次登录，关闭浏览器删除
local个人长期登录， 长期保存数据
9.堆和栈的理解
栈： 简单数据类型存放到栈里面，由操作系统自动分配释放存
放函数的参数值、局部变量的值等。
堆：存储复杂类型对象，一般由程序员分配释放，若程序员不释
放，由垃圾回收机制回收。
10.基础和复杂 的区别
存储上的区别
基本数据类型，直接存储在 栈 里
复杂数据类型，数据存储在 堆 里，把地址存储在 栈 里
11.深拷贝和浅拷贝的区别，怎么做一个深拷贝 和浅拷贝
浅拷贝是 遍历拷贝对象第一层属性名和属性值
深拷贝是遍历拷贝一个对象第一层属性名和属性值，如果发现属
性值是一个对象，则递归拷贝所有深层的属性

理),vue3实现有什么不同
vue2

缺点

调用强制更新）
要定义的数据必须要提前在data中定义好
vue3

13.闭包的理解，什么是闭包
简单理解：闭包 = 内层函数 + 外层函数的变量
闭包是一个 函数的执行空间,执行空间有一个局部变量被外部引
用，这个执行空间就是闭包
14.闭包使用场景
实现数据的私有

16.作用域链
作用域链本质上是底层的变量查找机制
在函数被执行时，会优先查找当前函数作用域中查找变量
如果当前作用域查找不到则会依次逐级查找父级作用域直到全局
作用域
17.垃圾回收机制
JS中内存的分配和回收都是自动完成的，内存在不使用的时候会
被垃圾回收器自动回收
两种回收算法
引用计数法
标记清除法
18.防抖和节流
防抖:如果在 n 秒内又触发了事件，则会重新计算函数执行时间
节流: 就是指连续触发事件但是在 n 秒中只执行一次函数
19.应用场景
防抖使用场景： 搜索框输入，设定每次输入完毕n秒后发送请
求，如果期间还有输入，则从新计算时间
节流使用场景： : 鼠标移动，页面尺寸发生变化，滚动条滚动等
开销比较大的情况下
